# 图片竞筛 — 竞品智能筛选

从 Amazon BSR 竞品标题和主图中，自动识别与自有商品**外观结构和功能相似**的对标竞品，过滤同品类但外观关键结构不一致的商品。

## 功能

- 输入自有商品标题，或上传自有商品主图（两者均可选，至少提供一项）
- 输入多个竞品标题，或上传任意多张竞品主图（每个竞品至少提供标题或主图）
- 调用 Qwen2.5-VL-3B 视觉模型逐个综合对比外观结构、功能和标题信息
- 展示匹配 / 排除结果、百分制匹配度分数及判断理由
- 支持按匹配度分数从高到低或从低到高排序
- 支持取消分析与单张重试

## 筛选规则

- 优先看商品是否“长得一样”或高度相似：整体形状、外轮廓、关键部件/附件、纹理和展示方式要接近
- 颜色不同可以算竞品
- 材质不再一票否决：外形和核心功能一致时，材质略有差异也可以算
- 功能完全不同不算；外观材质相近且功能只多或少 1～2 个小功能，可以算
- 参考图中出现的关键结构必须匹配，例如皮套、外罩、保护套、包边、套筒、外壳等

## 技术栈

- React 19 + TypeScript + Vite
- Tailwind CSS v4
- Qwen2.5-VL-3B（OpenAI 兼容 API）

## 快速开始

```bash
npm install
npm run dev
```

浏览器访问 `http://localhost:5173`。

## 环境变量

复制 `.env.example` 为 `.env` 并按需修改：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `VITE_API_BASE_URL` | API 基础地址 | `http://49.232.90.152:9896/v1` |
| `VITE_MODEL_NAME` | 模型名称（需与 `/v1/models` 一致） | `Qwen3-VL-4B` |
| `VITE_API_KEY` | API 密钥（如需要） | 空 |

## 构建与部署

```bash
npm run build
```

将 `dist/` 目录部署到任意静态托管服务（Nginx、OSS、Vercel 等）。

**注意：** 前端直连 API 需确保服务端已配置 CORS。若遇跨域问题，需在 API 服务端添加允许来源的响应头。

## 使用说明

1. 输入「我的商品标题」或上传「我的商品主图」
2. 输入「竞品标题」或上传多张「竞品主图」（可来自 BSR 排行榜截图）
3. 点击「开始筛选」，等待逐张分析完成
4. 在「匹配竞品」Tab 查看可对标竞品；「已排除」Tab 查看被过滤的非对标商品，并可按匹配度排序
   

## 示例场景

商品为 slow cooker（慢炖锅）时，BSR 前 50 可能包含：

- **匹配**：其他品牌的 slow cooker
- **排除**：liner（内胆）、pressure cooker（压力锅）、单独售卖的盖子或配件
